Spring学习整理（一）：Spring基础

2019-04-21
  • Java

一、Spring

Spring是一个开源框架，由Jack Johnson创建，是为了解决企业应用开发的复杂性而创建的。

Spring使用基本的JavaBean来完成以前只可能由EJB完成的事情。

Spring的应用不仅限于服务器端开发、从简单性、可测试性和松耦合性的角度而言，任何Java应用都可以从Spring中获益。

目的：解决企业应用开发的复杂性。

功能：使用基本的JavaBean代替EJB，并提供了更多企业应用功能。

Spring是一个容器框架，用来装JavaBean，中间层框架可以起到一个连接作用，比如把Struts2和Hibernate粘合在一起运用。简单来说，Spring是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器矿建。

在实际开发中，通常服务器采用三层体系架构，Spring对每一层都提供了技术支持。

表示层（Web）：在表示层提供了对Struts2等框架的整合。

业务逻辑层（Service）：在业务逻辑层可以管理事务，记录日志等。

持久层（Dao）：在持久层可以整合Hibernate和JdbcTemplate等技术。

二、Spring框架的优点

1.使用Spring的IoC容器，将对象之间的依赖关系交个Spring，降低组件之间的耦合性

2.可以提供众多服务，事务管理，WS等

3.AOP的很好支持，方便面向切面编程

4.对主流框架提供很好的集成支持，如Hibernate、Struts2、JPA等

5.Spring DI机制降低了业务对象替换的复杂性

6.低侵入，代码污染低

7.Spring的高度可开放性，并不强制依赖于Spring，开发之可以自由选择Spring的部分或全部

三、Spring的体系结构

Spring框架采用分层架构，其一系列功能要素被分为20个模块。
Spring的核心模块包含Core、Beans、Context和Expression Language四个小模块。其中，Core和Beans是整个Spring框架基础部分，也是Spring 的核心依赖注入IoC和DI的最基本实现，Spring的其他模块大多依赖这两个功能。

1.核心模块

spring-core：其他模块的基础核心，包含Spring框架的核心工具类，Spring其他模块都要使用该包里面的类。

spring-beans：Spring定义bean的支持，负责访问配置文件、创建和管理bean，支持依赖注入和控制反转的相关操作。Bean工厂类就在这个jar包中。

spring-context：Spring运行时容器，在Core和Beans的基础上，提供对Spring的上下文支持，ApplicationContext是该包的关键，通过它可以快速的取出依赖注入的bean。

spring-expression：Spring表达式语言，帮助Spring在运行时查询和操作对象。支持设置/获取对象的属性值，方法的调用。

2.AOP模块.

spring-aop：对于代理AOP的支持。

spring-Aspects：对于AspectJ的AOP支持。

3.Web模块

spring-web：提供基础的web功能，在web项目中提供Spring容器。

spring-webmvc：提供基于Servlet的SpringMVC。

spring-WebSocket：提供WebSocket功能。

spring-webmvc-portlet：提供portlet的支持。

4.数据库模块

spring-jdbc：提供jdbc访问数据库的支持，包含Spring对数据库访问操作进行封装的所有类，它提供了一个JDBC抽象层，从而实现对其他厂商的支持。

spring-tx：提供对事物的支持。

spring-orm：提供对对象关系映射的支持，使用Spring可轻松地整合其他第三方ORM库，如Hibernate、Mybatis、JAP等。

spring-oxm：提供对xml映射支持。

spring-jms：提供对java消息服务的支持。

四、Spring的核心容器

Spring框架的主要功能是通过Spring的核心容器来实现的。Spring容器会控制程序之间的关系，而不是由程序代码直接控制，Spring为我们提供了两种核心容器，BeanFactory和ApplicationContext。

1.BeanFactory

BeanFactory是一个基础类型的控制反转容器，是一个管理Bean的工厂，主要负责初始化各种Bean并调用他们的生命周期方法。
创建BeanFactory实例时，需要提供Spring所管理容器的详细配置信息，这些信息通常采用XML文件形式来管理。
加载配置信息的语法：

BeanFactory beanFactory =
new XmlBeanFactory(new FileSystemResource(“F:/application.xml”));

(这种加载方式开发中基本不用)。
BeanFactory是一个接口，我们使用其实现类（ XmlBeanFactory）来创建对象

2.ApplicationContext

ApplicationContext是BeanFactory的子接口，是另一种常用的Spring核心容器。其由org.springframework.context.ApplicationContext接口定义，不仅包含BeanFactory的所有功能，还添加了对国际化、资源访问、事务传播等哪方面的支持。

1.创建ApplicationContext接口实例

通常采用两种方法：

（1）通过ClassPathXmlApplicationContext创建

ApplicationContext applicationoContext = 
new ClassPathXmlApplicationContext(String configLocation);

ClassPathXmlApplicationContext会从类路径classPath中寻找指定的XML配置文件，找到并装在完成ApplicationContext的实例化。
此方法常用。

（2）通过FileSystemXmlApplicationContext创建

ApplicationContext applicationoContext = 
new FileSystemXmlApplicationContext(String cofigLocation);

FileSystemXmlApplicationContext会从指定的文件系统路径（绝对路径）中寻找指定的XML文件，找到并装在完成ApplicationContext的实例化。

2.Web项目中ApplicationContext的实例化

在java项目中，需要通过ClassPathXmlApplicationContext类来实例化ApplicationContext容器。

（1）实例化

在Web项目中，ApplicationContext容器的实例化工作会交给Web服务器来完成。
Web服务器在实例化ApplicationContext容器时，通常会使用ClassLoaderListener来实现，这种方式只需要在web.xml中添加如下代码：

<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>
        classpath:spring/applicationContext.xml
    </param-value>
</context-param>

<listener>
    <listener-class>
            org.springframework.web.context.ContextLoaderListener
    </listener-class>
</listener>

（2）Spring获取Bean实例

在Spring容器床创建之后，就可以获取Spring容器中的bean。Spring获取Bean的实例的两种方法：

Object getBean(String name);

根据容器中bean的id或name来获取bean，获取之后需要进行强制类型转换。

<T>T getBean(Class<T> requiredType);

根据类的类型来获取bean的实例，泛型方法，获取bean之后不需要强制类型转换。

3.BeanFactory与ApplicationContext区别与选用

两种容器都是通过XML文件配置和加载bean，两者主要区别在于，如果bean的一个属性没有注入，使用BeanFactory加载后，第一次调用getBean()方法时会抛出异常，而ApplicationContext则在初始化的时候自检，这样有利于检查所依赖的属性是否注入。因此，在开发中，通常优先使用ApplicationContext，而只有在系统资源较少时，才考虑使用BeanFactory。
